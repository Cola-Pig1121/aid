## 二、作品概述

`AID` 是一个面向普通用户的医疗报告智能分析作品，聚焦“看不懂体检报告、不会判断是否需要就医、不了解该去什么医院或科室”这一现实问题，尝试利用大语言模型、多模态理解、联网搜索与位置感知能力，构建一个可直接交互的智能医疗辅助系统。

作品提供三类核心能力：

1. 医疗报告图片分析
2. 健康问题智能问答
3. 本地化医院与科室推荐

系统当前已经实现可运行的 Streamlit 图形界面，并内置“演示模式”预置问题区，适合比赛现场快速展示；同时已打包生成 Windows 可执行文件 `dist/AID.exe`，可直接用于答辩和演示。

## 三、创作背景

在日常生活中，很多人拿到体检报告后，面对血常规、血脂、肝肾功能、心电图等专业术语，往往不知道异常指标意味着什么，也不知道下一步应该做什么。尤其是老人、学生家长以及缺少医学背景的普通用户，常常存在以下困难：

- 看不懂医学术语和化验指标
- 不清楚异常数值是否严重
- 不知道应该挂哪个科室
- 不了解本地医院的选择

传统解决方式主要依赖线下问诊，但存在排队时间长、获取帮助不及时、基础问题也要占用医疗资源等问题。因此，我希望设计一个能够“先帮助理解、再帮助判断、最后帮助引导”的 AI 辅助系统，让人工智能在医疗信息普及和就医引导方面发挥实际作用。

## 四、作品摘要

### 1. 主要特征

本作品采用自研的 ReAct Agent 工作流，由大语言模型负责推理决策，再根据用户需求调用联网搜索、位置获取、医院搜索、时间查询、记忆管理等工具，形成“分析 + 检索 + 推荐”的完整处理链路。对于上传的医疗报告图片，系统支持直接调用多模态模型进行图像理解，并给出中文分析结果。

### 2. 主要用途

作品主要用于帮助普通用户理解医疗报告、获取基础健康建议，并在需要就医时推荐本地相关医院和科室。它适合家庭健康管理、体检后辅助解读、常见医学问题咨询等场景，也可作为“人工智能 + 医疗辅助”方向的教学和展示作品。

### 3. 创新点

- 将 ReAct Agent 思维链与医疗辅助场景结合，不是单纯聊天，而是按任务分步骤调用工具。
- 对医学问题默认引入联网搜索校验，降低仅凭模型记忆回答带来的风险。
- 将位置感知能力引入医院推荐流程，增强回答的本地化和实用性。
- 同时支持文本问答与医疗报告图片分析，体现多模态人工智能能力。

## 五、功能说明

### 1. 医疗报告图片分析

用户可在底部输入框左侧点击 `+` 号上传体检报告或检查单图片，并可同时输入文字说明后一并发送。系统会把图片与用户文字组合成统一请求，调用多模态模型识别图像内容，提取其中的医学指标与关键信息，分析异常值可能代表的含义，并结合用户需求给出健康建议、复查建议和推荐就诊科室。

当前代码中，界面主流程由 [`src/ui/streamlit_app.py`](./src/ui/streamlit_app.py) 实现，图片分析采用“用户图文联合提交 -> 获取位置 -> 多模态分析 -> 搜索当地医院”的四阶段流程，并增加了适合现场展示的“演示模式”预置问题区。

### 2. 医学健康问答

用户可以直接输入健康相关问题，例如：

- 血压偏高要注意什么
- 窦性心动过缓是什么意思
- 总胆固醇偏高应该怎么调理

系统不是直接凭空作答，而是优先调用搜索工具获取外部信息，再结合模型进行归纳解释。这一流程由 [`src/agent/react_agent.py`](./src/agent/react_agent.py) 和 [`src/tool/search_tool.py`](./src/tool/search_tool.py) 配合完成。

### 3. 本地医院推荐

当用户询问“附近医院”“推荐心内科医院”等问题时，系统会先获取用户位置，再结合科室需求搜索本地医院信息，最后生成更有针对性的推荐结果。

相关实现位于 [`src/tool/location_tool.py`](./src/tool/location_tool.py) 和 [`src/tool/search_tool.py`](./src/tool/search_tool.py)。

### 4. 多模型兼容

系统支持两类兼容 OpenAI 接口的大模型服务：

- ModelScope
- OpenRouter

相关配置和切换逻辑位于 [`src/llm/client.py`](./src/llm/client.py) 与 [`src/main.py`](./src/main.py)。

### 5. 演示模式

为了便于比赛现场展示，系统在侧边栏加入了“演示模式”预置问题区，支持一键触发典型演示场景，包括：

- 体检报告解读
- 症状问答
- 医院推荐

这样在答辩或展示时，无需临时组织措辞，就可以快速演示作品的核心能力。

## 六、技术方案

### 1. 总体架构

本作品采用“界面层 + Agent 层 + 工具层 + 模型层”四层结构：

```text
用户输入 / 图片上传
        ↓
Streamlit 交互界面
        ↓
ReAct 医疗分析 Agent
        ↓
工具调用（搜索 / 定位 / 医院推荐 / 时间 / 记忆）
        ↓
大语言模型整合输出
```

### 2. 关键技术

| 模块 | 技术方案 | 作用 |
| --- | --- | --- |
| 界面交互 | Streamlit | 构建图形化交互界面与演示页面 |
| 智能体 | 自研 ReAct Agent | 负责推理、决策、工具编排 |
| 大模型接入 | OpenAI 兼容接口 | 支持 ModelScope / OpenRouter |
| 联网搜索 | Tavily | 查询医学信息与医院资料 |
| 定位能力 | IP 定位 + 腾讯地图逆地理编码 | 获取城市与详细位置 |
| OCR 解析 | pytesseract | 提供报告图片文字提取能力 |
| 打包发布 | PyInstaller | 生成可执行文件用于演示 |

### 3. 核心流程

#### 文本问答流程

1. 用户输入健康问题
2. Agent 判断是否需要联网搜索
3. 调用搜索工具获取外部信息
4. 兼容不同模型返回格式并提取可用文本
5. 汇总结果并生成中文回答

#### 图片分析流程

1. 用户在输入框左侧通过 `+` 号上传报告图片
2. 用户可补充文字说明并与图片一起发送
3. 系统获取当前位置信息
4. 多模态模型结合图片和文字共同理解用户需求
5. 结合分析结果搜索当地医院
6. 输出报告解读、身体状况分析、建议与医院推荐

## 九、运行与展示方式

### 1. 源码运行

```bash
uv venv
.venv\Scripts\activate
uv sync
python -m src.main
```

默认访问地址：

```text
http://localhost:7860
```

说明：

- 程序入口会自动启动 Streamlit Web 界面
- 图片通过输入框左侧的 `+` 号上传，可与文字一起发送
- 可通过 `--host` 与 `--port` 参数修改访问地址
- `--share` 参数仅为兼容保留，当前 Streamlit 模式下不会启用公网分享

### 2. 环境变量配置

首次运行前需要配置 `.env` 文件。项目已提供模板 [`.env.example`](./.env.example)。

核心配置项如下：

```env
MODELSCOPE_API_KEY=your_key
OPENROUTER_API_KEY=your_key
TAVILY_API_KEY=your_key
TENCENT_MAP_KEY=your_key
TENCENT_MAP_SK=your_sk
```

### 3. 可执行文件展示

项目已生成可执行文件：

- [`dist/AID.exe`](./dist/AID.exe)

该文件可作为现场展示版本使用。启动后会直接运行 Streamlit 界面，适合在比赛电脑上进行快速演示。

## 十二、当前不足与后续改进

目前作品仍有继续提升的空间：

- 医疗报告的结构化抽取能力还可以进一步增强
- OCR 能力对不同版式报告的适应性还需优化
- 医院推荐结果依赖外部搜索接口，稳定性受网络影响
- 尚未形成系统化测试用例与量化评估报告
- 未来可增加语音交互、健康档案管理和随访提醒功能

后续计划围绕“更准的报告理解、更强的本地医疗推荐、更适合老年用户的交互方式”继续完善。

## 十三、结论

AID 医疗报告智能分析系统是一次人工智能技术面向民生场景的实践探索。它尝试将大语言模型、工具调用、多模态理解和位置感知能力结合起来，帮助普通用户更容易理解体检和检查信息，并在需要时获得更明确的就医方向。

本作品既体现了人工智能编程的技术实现能力，也体现了面向实际问题设计解决方案的思考，符合“人工智能（编程）”赛项对作品完整性、创新性和应用价值的要求。
